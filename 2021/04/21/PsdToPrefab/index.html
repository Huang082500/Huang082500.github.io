PSD To Prefab

需求来源

最近项目中会涉及大量的 UI 修改，传统的 UI 做法是美术同学出效果图 （PSD）和对应的图素，并用 TexturePacker 工具导成 Unity 中的图集资源，然后程序同学或者美术同学来拼 Unity 中的 UI,程序同学再补充 UI 的相关逻辑，为了减少在 Unity 中重复制作 UI 的步骤，达到缩短开发流程的目的， 需要将美术的 PSD 文件直接导成 UnityPrefab 资源。

需求分析

  1. 需要给制作 PSD 的美术同学制定制作 PSD 的规范，包括层次结构和命名规则。
  2. 根据上一步来将生成的 PSD 按照既定的制作规范，来导出序列化资源。
  3. 将导出的序列化资源导入到 Unity 中，在生成对应的 Prefab 之前，需要将 PSD 中引用到图片使用 TexturePacker 进行构建。
  4. 将生成的 Prefab 中所引用的图素，更换成 Unity 项目中所使用的图素。

开发环境

MacOS 14.5

Unity2018.4

Python3.9

PSD To Prefab Tool: https://github.com/kyubuns/Baum2

需求实现步骤

具体的 Psd To Prefab 工具在网上可以找到很多，包括在 AssetStore 上也有一些付费的工具。我在这使用的是开源的工具 Baum2 : https://github.com/kyubuns/Baum2。具体的步骤在该链接里都有介绍。我详细说一下需求分析第三步该怎么做。
