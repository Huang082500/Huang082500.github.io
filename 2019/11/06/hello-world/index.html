使用Hugo搭建静态网页

 Posted on November 5, 2019  |  1 minutes  |  111 words  |  黄圣博

部署我的服务器流程

  1. 首先在GitHub上注册我的账号名称：huangshengbo5
  2. 接着建造我的第一项目：http://huangshengbo.github.io/
  3. 项目建造完以后，点击设置（setting) ,跳转界面后，看到GitHubPages界面下的ChangeTheme按钮，点击进去可以选择自己中意的网页面板。
  4. 最重要的来了，就是将自己的域名和当前的GitHub工程联系在一起。首先需要在namesiol里注册账号(www.namesoil.com),然后直接选择输入自己想要的域名，一般是自己名字的英文翻译。庆幸 huangshengbo.com 这个域名还在。
  5. 接着上一步说，在我们在购买完属于自己的域名以后，在namesoil页面中点击domain manager 按钮，跳转界面后点击自己的域名按钮，选择DNS Records 左侧的Update 按钮，再次跳转界面后，在页面的中部可以看到自己域名的一些信息，选择Edit按钮，可以在Target Hostname中填写自己在GitHub中已经建好的工程名称。点击Submit按钮，就可以将自己的工程重定向到自己的域名中。

如何将本地的工程加载到GitHub上

  1. 首先需要去 https://git-scm.com 网站上下载最新的Git插件，下载好后安装运行。

  2. 如果是先在GitHub上建造了一个新的仓库，就会直接出一个界面需要你将一个本地的工程文件重定向到当前创建的GitHub库中。 实例代码：

    git remote add origin https://github.com/huangshengbo5/Test.git
git push -u origin master

  3. 在自己建立的本地目录下邮件可以看到GitBaskHere选项，点击会弹出一个命令行窗口，将上面的实例代码拷贝到命令行中，点击回车就将本地的目录重定向到自己的GitHub仓库中了。

如何添加环境变量

  1. 这是Hugo的下载地址 https://github.com/gohugoio/hugo/releases 我的电脑是windows64位，所有我下载了 hugo_0.59.1_Windows-64bit.zip 版本。
  2. 我现在的需求是将 hugo 这个变量加到电脑的环境变量中。
  3. 首先右击我的电脑，点击属性，点击右侧的高级系统设置，点击弹窗下面的环境变量，点击下面的系统变量的新建按钮，在 Name 一栏中填写自己想要的变量名称，我起的是 hugo ,下面的 path 路径设置为当前下载的 Hugo.exe 可执行文件的目录的路径，点击确定。
  4. 最后还要在上述的环境变量的 path 变量中添加 Hugo.exe 可执行文件的目录的路径，方法是将文件目录路径加到 path 变量的前面，并且用 ; 分开。点击确定，就完成了。

使用Hugo搭建静态网页

  1. cmd 命令： hugo new site myblog myblog :我的目录名称
  2. 可以下载一个Theme主题： git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke;
  3. hugo new site blog :我的一篇博客名称。
  4. 可以直接建立本地的Http服务器： hugo server -t ananke –buildDrafts
  5. 设置本地的baseUrl:hugo –baseUrl=“https://huangshengbo.com
  6. 在第六步后会在本地的mublog文件夹下创建一个publish的文件夹。
  7. 在新的publish文件下操作。将博客关联到GitHub上：git remote add origin http://github.com/huangshengbo5/huangshengbo5.github.io.git

  8. 如果想要重新生成Public（静态网页设置）直接在Public父目录中使用Git 执行

    hugo

  9. 每次在本地执行Git后会进入Vim编辑器中，这时点击 i 可以进入编辑器模式，点击 esc 可以退出编辑器模式，进入命令行模式，点击shift+: 接着输入w q 就可以退出vim编辑器。此时的vim编辑器只是让你可以给该次的提交添加一下log。

  10. cd .. 去上一层目录

  11. 在每一篇的文章中最好不要有draft : true 标记，这表明该文章是一篇草稿。在生成的静态网页中，无法被看到。
